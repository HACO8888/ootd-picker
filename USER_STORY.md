# USER_STORY — OOTD Picker

> 分組討論收斂後的使用者故事。參與角色：PM、軟體、架構、設計。
> 本文件為需求源頭，PRD 與驗收標準皆回溯至此。

## 1. 專案題目（一句話）

> 「讓使用者每天早晨用 30 秒，依**心情／天氣／目的地／性別**得到一套完整的穿搭＋妝容＋香水建議，並能管理自己的膠囊衣櫥與收藏喜歡的組合。」

## 2. 目標使用者（Personas）

| Persona | 描述 | 痛點 | 期待 |
|---|---|---|---|
| **上班族 — 小庭（女, 27）** | 通勤族，衣櫥不大但講究質感 | 早上選衣服花時間、不知妝容怎麼搭 | 快速得到「可直接照做」的整套建議 |
| **學生 — 阿宏（男, 21）** | 預算有限，想看起來得體 | 不懂香水/理容、怕踩雷 | 中性、低門檻的清爽建議 |
| **極簡愛好者 — Lin（不分性別, 30）** | 信奉膠囊衣櫥、重視永續 | 不想衝動購物、想最大化現有單品 | 用既有單品換出多種搭配 |

## 3. 史詩（Epics）

- **E1 風格嚮導**：依條件生成今日整體造型。
- **E2 膠囊衣櫥**：管理單品（瀏覽／篩選／新增／刪除）。
- **E3 收藏**：保存與回顧喜歡的組合。
- **E4 體驗與資訊**：首頁行銷、關於理念、跨裝置體驗。
- **E5 迭代工具**：天氣偵測、衣櫥洞察、穿搭日誌與分享。
- **E6 帳號與雲端同步**：Google 登入，衣櫥/收藏/日誌跨裝置同步。
- **E7 管理員後台**：管理用戶、目錄、妝容香水與內容稽核。

## 4. 使用者故事與驗收條件

格式：**As a** 〔角色〕**, I want** 〔功能〕**, so that** 〔價值〕。

### E1 風格嚮導

- **US-01** As a 使用者, I want 用 4 步驟（性別→天氣→心情→目的地）輸入今天的情境, so that 不必填表單也能快速開始。
  - ✅ 進度條與步驟點隨選擇前進；可「返回上一步」修改。
  - ✅ 第 4 步後顯示載入動畫，再呈現結果。
- **US-02** As a 使用者, I want 得到一套包含**上衣／下著／外套（視天氣）／配件（有合適者）／妝容／香水**的完整建議, so that 我可以直接照著穿搭與化妝。
  - ✅ 寒冷/雨天有外套候選時必出外套；不合適的配件可留空，避免硬湊。
  - ✅ 妝容依性別切換「妝容／理容」標題。
  - ✅ 結果頁頂端以標籤回顯本次的性別/天氣/心情/目的地。
- **US-03** As a 使用者, I want 對任一單品/妝容/香水單獨「換一個」或整體「重新生成」, so that 不滿意時能微調而不必重來。
  - ✅ 換單品時優先在符合情境與季節的同類中挑選，並考量目前整套搭配；無可換時提示。
- **US-04** As a 阿宏（男）, I want 系統推薦中性或男性向的妝容與香水, so that 建議符合我的需求。
  - ✅ 妝容/香水依 `gender` 過濾與加權；純男性情境只顯示男性理容候選。

### E2 膠囊衣櫥

- **US-05** As a 使用者, I want 瀏覽我的全部單品並看到品牌/季節/配色/類別標示, so that 一眼掌握衣櫥。
- **US-06** As a 使用者, I want 依**類別／季節／品牌／標籤／配色**篩選與**關鍵字搜尋**, so that 快速找到特定單品。
  - ✅ 多條件可組合；無結果顯示空狀態與「重設篩選」。
- **US-07** As a 使用者, I want 上傳新單品（可拖放圖片、填寫名稱/類別/品牌/季節/主色/標籤）, so that 衣櫥反映我真實擁有的衣服。
  - ✅ 未上傳圖片時，依名稱與類別**自動配對**代表圖。
  - ✅ 必填驗證：名稱、至少一季節、主色。
- **US-08** As a 使用者, I want 刪除單品（含確認）, so that 移除已不穿的衣服。

### E3 收藏

- **US-09** As a 使用者, I want 收藏目前的整體組合, so that 日後可回顧或重現。
- **US-10** As a 使用者, I want 在任一頁開啟收藏抽屜檢視/刪除, so that 隨時管理我的收藏。
- **US-11** As a 使用者, I want 把某個收藏「載入至畫布預覽」, so that 直接在嚮導結果頁重現該組合。
  - ✅ 由其他頁觸發時自動導向 `/picker` 並載入。

### E4 體驗與資訊

- **US-12** As a 新訪客, I want 在首頁了解產品價值與運作方式, so that 決定是否開始使用。
- **US-13** As a 使用者, I want 在手機與桌機都有良好版面與底部導覽, so that 隨處可用。
- **US-14** As a 回訪使用者, I want 我的衣櫥與收藏在重新整理後仍在, so that 不必每次重建。
  - ✅ 以 `localStorage` 持久化（key 與舊版相容）。

### E6 帳號與雲端同步（第 6 輪）

- **US-15** As a 使用者, I want 用 Google 一鍵登入/登出（且不登入也能用）, so that 我可以選擇要不要綁定帳號。
  - ✅ 頂部 `UserMenu`；訪客路徑完全不受影響（可選登入）。
- **US-16** As a 回訪/換裝置的使用者, I want 登入後我的衣櫥/收藏/日誌跨裝置同步, so that 換手機或瀏覽器資料都在。
  - ✅ 登入時 union by id + last-write-wins 合併，不覆蓋訪客既有資料；之後每次變更自動同步上雲。
- **US-17** As a 極簡愛好者 Lin, I want 登入前在本機建立的資料，登入後完整保留並上雲, so that 不會因為登入而遺失。
  - ✅ `mergeOnLogin` 合併本地與雲端，無重複、無遺失。

### E7 管理員後台（第 6 輪）

- **US-18** As a 管理員, I want 管理用戶（升降權/停權/刪除/查看資料）與查看數據統計, so that 能維運平台與掌握使用概況。
  - ✅ `role=admin`（`ADMIN_EMAILS` 白名單登入時 bootstrap）才可進 `/admin`；非 admin 被擋。
- **US-19** As a 管理員, I want 管理目錄（全域服裝/覆蓋/妝容/香水）並稽核使用者上傳內容, so that 維持內容品質。
  - ✅ 妝容/香水/目錄 CRUD 即時反映前台；上傳衣物 pending → 通過/拒絕，被拒不公開顯示。

## 5. 不在本輪範圍（Out of Scope）

- 線上購買/電商導購。
- Email／密碼註冊、其他社群登入（本輪僅 Google）。
- 推播通知、多人協作衣櫥。

## 6. 收斂結論

第 1～5 輪以 **E1～E5 的 US-01～US-14** 為 MVP 與迭代範圍；**第 6 輪**新增 **E6～E7 的 US-15～US-19**（帳號、雲端同步、管理員後台）。US 對應的 ✅ 條件即為驗收依據，彙整於 [AGENTS.md](AGENTS.md) 的「驗收標準」與 [PRD.md](PRD.md)。
